Targets_Workshop

Establishing Pipilines with targets in R

Introduction

What you will learn and why

Why you should use Targets:

  • maintain a reproducible workflow
  • save costly runtime for tasks that are already up to date
  • provide evidence that output aligns with the code and data

What you will learn in this workshop:

  • How to build a pipeline of custom functions using tar_script() , tar_edit()and tar_target()
  • How to visualize dependencies using tar_visnetwork() & tar_manifest()
  • How to run your pipeline using tar_make()
  • How to check for outdated targets using tar_outdated()
  • How to access targets to work with them using tar_read()and tar_load()
  • How to monitor data files for changes by establishing them as targets

A pipeline of functions, the Targetsâ€™ Philosophy

A pipeline contains tasks to prepare datasets, run models, and summarize results for a business deliverable or research paper. The methods behind these tasks are user-defined R functions that live in R scripts, ideally in a folder called â€œ3-R/â€ in the project. The tasks themselves are called â€œtargetsâ€, and they run the functions and return R objects. The targets package orchestrates the targets and stores the output objects to make your pipeline efficient, painless, and reproducible. -Source

Building a pipiline of custom functions

Functions

The commands of your targets will depend on the functions from 1-functions.Rmd. For the exercises below, they are in 3-R/1-functions.R. Take a quick glance at 3-R/functions.R familiarize yourself with the code base.

Start with a small pipeline.

The first version of the pipeline consists of three targets, or steps of computation. Our first targets do the following:

  1. Reproducibly track the NYC flights data file.
  2. Read in the data from the CSV file.
  3. Build a simple linear model predicting delays.
  4. Plot the model
  5. Report results

These steps are defined as custom functions in the 1-functions.Rmd

The _targets.R file

To initiate our targets, we express the computation in a special configuration file called _targets.R which has to be at the projectâ€™s root directory. This file is necessary to:

  1. Load any functions and global objects that the targets depend on.
  2. Set any options, including packages that the targets depend on.
  3. Declare a list of targets using tar_target(). The _targets.R script must always end with a list of target objects.

You would create this file from scratch using the tar_script() command.

We have prepared and initial file for you here which you will edit throughout the workshop. Run the following code chunk to put our prepared _targets.R file for this chapter in your working directory.

file.copy("3-R/2-initial_targets.R", "_targets.R", overwrite = TRUE)

Data files as targets

In the targets scrip you can also specify a file as a target. In fact that is what we did with the NYC-Flights data set with format = "file" in tar_target(). This way any changes to the data set should trigger the target and all targets that depend on it to become outdated. tar_read() returns the path for these types of targets [targets with ].
